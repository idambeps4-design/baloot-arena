# Balot Arena v1.1.0

تطبيق عربي RTL قابل للتثبيت على iPhone مبني بـ Next.js وSupabase لحسبة البلوت، إدارة اللاعبين، المنافسات، الإحصائيات وملخص نهاية الصكة.

راجع `RULEBOOK.md` لقواعد الحسبة المعتمدة، و`DEPLOY_FROM_SCRATCH.md` للنشر الكامل، و`VERIFICATION.md` لنتائج التحقق الفعلية.

## أهم خصائص v1.0

- عند اختيار الطالع، يتحول إدخال عدد الورق تلقائياً إلى الفريق المقابل، ويمكن تغييره يدوياً.
- في حكم، آخر مكبر يصبح الطالع الفعلي، والتعادل في التكبير يُحسب خسارة له.
- في صن وأشكل، التكبير يتوقف عند دبل ويُتاح فقط عندما يكون فريق الطالع 100 أو أكثر والخصم أقل من 100.
- في دبل وثري وفور يحصل الفائز فقط على الأساس والمشاريع المضاعفة، بينما يبقى بلوت بنقطتين بدون مضاعفة.
- في كبوت وكبوت عكسي تُضاف مشاريع الفريق الفائز إلى قيمة الكبوت الثابتة، ويبقى بلوت بنقطتين.
- قهوة تحسم الراوند 152–0.
- الصكات غير المكتملة لا تُحفظ ولا تُستعاد. الحفظ يتم مرة واحدة بعد انتهاء الصكة فقط.
- الحفظ النهائي ذري داخل Supabase: الصكة والراوندات تنجح معاً أو لا يُحفظ أي جزء.
- مفتاح فريد لكل صكة يمنع تكرار الصكة المكتملة عند إعادة الضغط أو إعادة المحاولة.
- ملفات لاعبين تشمل الفوز والخسارة، النسبة، السلاسل، الطلبات، الكبوت، القهوة، المشاريع، الشركاء والخصوم.
- ملخص عربي نهائي مع نجم الصكة، مفلس الصكة، أسباب الاختيار، قصة مختصرة للأحداث وبحد أقصى نكتتين متوافقتين مع النتيجة.
- إعلانات وسط الشاشة للأحداث المهمة، ومنها إعلان «زخه التفتيش» عند خسارة فريق عبدالله شريف للراوند أو الصكة، مع منعه عند الفوز وزر إخفاء.
- الترتيب وملفات اللاعبين تتحدث فور حفظ الصكة ثم تتطابق مع بيانات Supabase.
- لا توجد صفحة تاريخ مباريات كاملة أو خط زمني عام للراوندات. تظهر فقط آخر خمس نتائج في الرئيسية وراوندات الصكة الحالية لغرض الحسبة والتعديل.


## PWA وiPhone

- اسم التطبيق المثبت: **Balot Arena**.
- يعمل بوضع مستقل بدون شريط Safari بعد إضافته إلى الشاشة الرئيسية.
- يتضمن أيقونات 180 و192 و512 ونسخة maskable.
- يحترم Dynamic Island وHome indicator عبر safe areas.
- لا يخزن طلبات Supabase أو API في الكاش، لذلك لا يعرض ترتيباً قديماً بسبب الـPWA.
- تظهر رسالة واضحة عند انقطاع الإنترنت لأن الحفظ والتحديث يحتاجان اتصالاً.

راجع `PWA_INSTALL.md` لخطوات الرفع والتثبيت على iPhone.

## تشغيل محلي

```bash
npm ci
cp .env.example .env.local
npm run dev
```

ثم ضع بيانات Supabase العامة في `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_xxxxxxxxx
```

لا تستخدم `service_role` أو أي مفتاح سري.

## إعداد Supabase

### مشروع Supabase جديد

شغّل الملف الكامل التالي في **SQL Editor**:

```text
supabase/setup.sql
```

### قاعدة ساحة البلوت موجودة مسبقاً

شغّل ترقية v1.0 فقط:

```text
supabase/v1.sql
```

الترقية تضيف:

- `matches.match_key`
- `matches.summary_json`
- فهرساً فريداً لمنع تكرار الحفظ
- الدالة الذرية `save_completed_match_v1`
- حذف الصفوف القديمة غير المكتملة أو الملغاة

## التحقق

```bash
npm run verify
```

يشغّل بالترتيب:

1. فحص `.npmrc` ومصادر `package-lock.json`.
2. اختبارات الحسبة والإحصائيات والملخص.
3. فحص TypeScript.
4. بناء الإنتاج.

راجع `VERIFICATION.md` لنتيجة التحقق الخاصة بهذه الحزمة.
